Requisitos No Funcionales

Funcionalidades

En estos requisitos se observarán algunos requisitos que pueden considerare funcionales. Como lo son la Adecuación y la Seguridad de Acceso.

FUNCIONALIDADES


Adecuación

En el rol de administrador se proporciona un conjunto apropiado de funciones específicas a dicho rol.


Seguridad de acceso

Si algún usuario desea ingresar con un rol el cual no le pertenece su acceso será denegado así protegiendo la información de los demás roles.

Usabilidad

Este tipo de requisitos contiene las capacidades para ser entendido, aprendido y usado.

USABILIDAD


Capacidad para ser entendido

El software cuenta con la capacidad de permitir al usuario entender si el software es adecuado y como puede ser usado.


Capacidad para ser aprendido

El software es intuitivo y fácil para el usuario aprender su uso y/o manipulación.


Capacidad para ser operado

Al usuario le serán fácil hacer uso del aplicativo sin perdida.

Calidad de Uso

Aquí observaremos la efectividad, la seguridad física y la satisfacción del software.

CALIDAD DE USO


Efectividad

El software está diseñado para suplir la necesidad de crear horarios de forma precisa y rápida por lo cual estará enfocado en esta necesidad siendo así muy efectivo.


Seguridad física

Nuestro software no tiene posibilidad alguna de dañar físicamente al usuario a la empresa o al hardware.


Satisfacción

La forma de observar sus horarios será más práctico y fácil por lo cual les dará la satisfacción de hacer uso de ella.

Eficiencia

La eficiencia se asegura del comportamiento temporal y del cumplimiento de la eficiencia del software.

EFICIENCIA


Comportamiento temporal

El software tendrá una respuesta rápida y efectiva.


Cumplimiento de la eficiencia

El software será muy eficiente cumpliendo con las normas básicas de la eficiencia sin importunar al usuario y dándole respuesta rápida a sus procesos.

Mantenibilidad

Aquí daremos a conocer la capacidad para ser analizado.

MANTENIBILIDAD


Capacidad para ser analizado

El usuario podrá observar si hay alguna anormalidad en el sistema.

Portabilidad

Se conocerá la adaptabilidad y su cumplimiento.

PORTABILIDAD


Adaptabilidad

El usuario podrá observar si hay alguna anormalidad en el sistema.


Cumplimiento de la portabilidad

El software puede adherirse a normas y convenciones relacionadas con portabilidad.
